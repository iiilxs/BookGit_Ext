BookGit 帮助指南

通过 GitHub 仓库同步 Chrome 浏览器书签，无需 VPN，无需服务器

🚀 快速开始

按以下 5 步操作，10 分钟完成配置。

Step 1 · 注册 GitHub 账号

如果没有 GitHub 账号，先去注册一个。
→ 打开 github.com 注册

Step 2 · 创建私有仓库

登录后创建一个仓库，用来存你的书签数据。

  1. 点右上角 + → New repository（新建仓库）
  2. 仓库名填写 bookgit（任意都行）
  3. 一定要选 Private（私有，别选 Public），书签是隐私数据
  4. 不要勾任何初始化选项，直接点创建

→ 打开 github.com/new 创建仓库

Step 3 · 生成 Token（访问密钥）

Token 相当于密码，让插件可以读写你的仓库。

  1. 点右上角头像 → Settings（设置）
  2. 左下角 Developer settings（开发者设置） → Personal access tokens（个人访问令牌） → Fine-grained tokens（细粒度令牌）
  3. 点 Generate new token（生成新令牌）
  4. Token name 写 BookGit
  5. Repository access（仓库访问） → 选 Only select repositories（仅选定仓库） → 勾选你刚创建的仓库
  6. Permissions（权限） → Repository permissions（仓库权限） → 找到 Contents（内容） → 选 Read and write（读写）
  7. 点 Generate 生成
  8. 立即复制 Token！页面关了就看不到了

→ 打开 GitHub Token 设置页

Step 4 · 填入插件

点击浏览器工具栏的 BookGit 图标，在配置面板填入：

  • Token：粘贴刚复制的 Token，点 🔒 保存
  • 所有者：自动填充为你的 GitHub 用户名
  • 仓库：下拉选择你创建的仓库
  • 分支：默认 main 即可
  • 路径：保持默认 bookgit.json
Step 5 · 测试连接并同步

点 ● 测试 按钮验证配置。

  • 成功 → Token 自动锁定，无法再查看修改
  • 失败 → 看错误提示，检查 Token 和仓库权限

测试通过后，点 ↻ 立即同步 → 选择 🔄 智能合并，书签就开始同步了。

🔄 三种同步方式

点击「立即同步」后选择：

智能合并 推荐
两边书签合并，URL 自动去重。日常用这个就行。
上传本地 覆盖
用本地书签覆盖远程，远程数据会被替换。
远程下载 覆盖
用远程书签覆盖本地，适合新电脑或误删恢复。

📖 功能介绍

定时同步

开启「自动同步」后，插件按设定间隔自动执行智能合并，两端始终一致。

Token 安全锁

Token 一旦验证通过，立即永久锁定。之后无法查看、无法修改，彻底防止泄露。如需更换，需在扩展管理页清除存储后重新配置。

一键去重

扫描本地书签中 URL 重复的条目，保留首次出现的位置，帮你清理冗余书签。

修复远程数据

下载远程书签 → 自动去重 → 重新上传。不影响本地书签，专门修复 GitHub 上已积累的重复项。

导出 / 导入配置

「导出」将仓库设置复制到剪贴板，不含 Token。换电脑时「导入」即可恢复配置，只需重新填入 Token。

❓ 常见问题

两台电脑书签不同，同步后会怎样？

智能合并会保留两边的书签，同一网址不重复添加。合并后两台电脑内容完全一致。

误删了书签能恢复吗？

如果另一台电脑同步过，从那边选「远程下载」即可。如果两台都误删了，登录 GitHub 从 bookgit.json 的提交历史中找回。

Token 忘记保存了怎么办？

回到 GitHub Token 设置页，删除旧 Token，重新生成一个新 Token。在插件中填入新 Token 即可。

插件加载后没反应？

在 chrome://extensions 找到 BookGit，点卡片右下角的刷新按钮（⟳），重新加载一次。
